# Azure 报价学习与实操手册

这是一套面向 **新手 / 销售支持 / 初级云报价同学** 的 Azure 报价入门资料。

这套文档不是讲很深的云架构原理，也不是官方产品白皮书，而是专门解决一个最实际的问题：

> **客户给了一个业务需求，我们怎么把它翻译成 Azure 资源，并整理成可落地的报价思路。**

---

# 这套资料适合谁看

适合以下人群：

- 刚接触 Azure 报价的新手
- 销售、售前、商务支持人员
- 需要看懂报价单但技术基础不深的同学
- 想快速建立 Azure 资源概念的人

如果你现在也有下面这些困惑，这套资料会比较适合你：

- 不知道 Azure 报价到底在报什么
- 看见 VM、磁盘、Redis、CDN、Defender 不知道是干嘛的
- 知道客户要“上系统”，但不会拆成云资源
- 会查价格，但不知道哪些资源该报、哪些容易漏
- 不清楚 CPU / GPU、D4as v5 / D8as v5 这些规格怎么理解

---

# 这套资料的核心目标

看完后，你应该能做到下面几件事：

1. **理解 Azure 报价本质**
   - 不是报“服务器价格”
   - 而是报“客户系统在云上运行的整体成本”

2. **能把客户需求翻译成 Azure 资源**
   - 例如把“官网 / APP 后台 / 电商系统 / 管理系统”拆成：
   - 计算、存储、数据库、网络、安全、备份等资源项

3. **能看懂常见配置项**
   - 如 VM 规格、CPU/内存、系统盘/数据盘、Redis、CDN、负载均衡、备份等

4. **能形成基础报价流程**
   - 收集需求
   - 判断资源类型
   - 拆分资源项
   - 查价
   - 汇总
   - 加备注说明

5. **能避免常见漏项**
   - 比如磁盘、备份、带宽、负载均衡、安全项等

# 这套资料的学习方法

不建议一开始死记价格，也不建议一上来就背所有产品。

更好的方法是按下面 3 步来学：

## 第一步：先认识资源是干嘛的
例如：

- VM：跑程序
- Disk：放系统和数据
- MySQL / SQL：存业务数据
- Redis：做缓存
- CDN：加速静态内容
- Load Balancer：分流
- Backup：备份
- Defender / Firewall：安全

---

## 第二步：练会“需求 → 资源”
比如客户说一个“电商系统”，你能想到：

- 应用服务器
- 数据库
- 存储
- 缓存
- 负载均衡
- 安全
- 备份

---

## 第三步：练会“资源 → 报价单”
也就是把资源项列清楚后，再去查：

- 区域
- 规格
- 操作系统
- 付费方式
- 月度 / 年度成本

---

# 这套资料默认的思考框架

以后看到一个 Azure 报价需求，建议统一按下面 5 类去拆：

## 1. 计算
程序跑在哪里？

常见资源：
- VM
- App Service
- AKS
- Function

## 2. 存储
文件和磁盘放在哪里？

常见资源：
- Managed Disk
- Blob Storage
- Azure Files

## 3. 数据库
业务数据放在哪里？

常见资源：
- Azure SQL
- MySQL
- PostgreSQL
- Redis

## 4. 网络
用户怎么访问系统？

常见资源：
- Public IP
- Load Balancer
- VPN
- CDN
- Firewall

## 5. 安全与备份
怎么防攻击、怎么恢复？

常见资源：
- Defender
- Backup
- DDoS
- Key Vault

# 使用这套资料时要注意

## 1）这不是 Azure 官方定价承诺
本仓库内容主要用于：

- 学习
- 内部培训
- 报价思路整理
- 新人上手

正式报价时，仍应以 Azure 官方价格页、计算器、合作伙伴价格政策、实际折扣政策为准。

---

## 2）价格会受多种因素影响
Azure 价格通常受以下因素影响：

- 区域（Region）
- 机型规格
- 操作系统（Linux / Windows）
- 计费模式（按量 / 1年预留 / 3年预留）
- 实际使用量
- 汇率与政策变动

---

## 3）很多费用不是固定死价
例如下面这些通常需要按实际估算或说明：

- 出网流量
- 存储实际占用量
- 快照 / 备份容量
- CDN 流量
- 安全防护等级
- 数据库性能层级

---

## 4）报价单不是架构设计书
这套资料主要帮助你：

- 看懂需求
- 拆对资源
- 少漏项
- 提高沟通效率

如果遇到复杂场景，例如：
- 双活容灾
- 大规模数据库集群
- 专线接入
- 多区域部署
- 大模型 / GPU 推理
- 金融 / 医疗强合规架构

建议仍由资深售前或架构师参与确认。

# 一个最简单的记忆法

你可以把 Azure 报价先记成一句话：

> **算力 + 存储 + 数据库 + 网络 + 安全/备份 = 报价主体**

翻译成人话就是：

- 程序跑在哪
- 数据放在哪
- 文件放在哪
- 用户怎么访问
- 出问题怎么防

只要你能用这 5 类去看客户系统，报价就会越来越清晰。
