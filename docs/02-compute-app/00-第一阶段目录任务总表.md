# 第一阶段目录任务总表

## 1. 阶段说明

第一阶段是整个 Azure 学习路径里的基础阶段。  
这个阶段的核心目标不是把服务学全，而是先把最常见、最基础、最容易在后续反复遇到的概念和实验打通。

这一阶段重点围绕下面几类内容展开：

- Azure 基础概念
- 订阅、资源组、区域
- Windows / Linux VM
- RDP / SSH 远程连接
- VM 常见问题与排障
- NSG 基础规则
- VNet / Subnet / Public IP 基础理解
- Blob Storage 与 SAS Token
- Azure OpenAI 基础部署体验

可以把这一阶段理解成：

> 从“刚接触 Azure”到“能独立完成基础资源创建、访问测试和简单排障”的入门阶段。

---

## 2. 阶段目标

完成这一阶段后，理想状态是达到下面这些目标：

### 2.1 理解 Azure 最基础的资源管理逻辑
包括：

- Subscription
- Resource Group
- Region

### 2.2 能独立创建最常见的基础资源
例如：

- Windows VM
- Linux VM
- Storage Account
- Blob Container
- Azure OpenAI 资源

### 2.3 能完成基础访问测试
例如：

- RDP 登录 Windows VM
- SSH 登录 Linux VM
- 使用 Blob URL 访问文件
- 使用 SAS URL 做临时访问测试
- 在 Playground 测试 Azure OpenAI

### 2.4 能做基础问题排查
例如：

- 连不上 VM
- NSG 没放通
- Blob URL 打不开
- SAS 过期
- Azure OpenAI 模型部署失败或调用失败

### 2.5 建立最基础的客户表达能力
例如能说清：

- Azure 是什么
- 订阅和资源组是什么
- Region 为什么重要
- VM 和网络对象之间的关系
- Blob Storage 和 SAS 是干什么的
- Azure OpenAI 基础上能做什么

## 8. 第一阶段的重点不是“学全”，而是“打通主线”

这一阶段最容易犯的错误是：

- 一会儿学 VM
- 一会儿学数据库
- 一会儿学 AI
- 一会儿又学网络
- 最后每样都只停留在表面

所以建议这一阶段先抓住主线：

> 资源管理 → 计算 → 网络 → 存储 → 最基础 AI 体验

只要这条线打通，后面第二阶段、第三阶段再扩展时会轻松很多。

## 10. 第一阶段实践建议

学习这一阶段时，建议坚持下面几个动作：

### 10.1 每篇至少动手做一次
不要只看文档，至少亲手点一遍 Portal 或执行一次测试。

### 10.2 每篇记录失败现象
例如：

- 为什么 SSH 连不上
- 为什么 Blob URL 打不开
- 为什么 SAS 过期后不能访问
- 为什么模型部署看不到

这些记录比“成功截图”更有价值。

### 10.3 每篇都用自己的话复述一次
这样你会更快从“抄笔记”进入“能讲清楚”。

### 10.4 不要忘记清理资源
尤其是：

- VM
- Public IP
- Disk
- Storage Account
- AI 资源

避免实验资源长期闲置产生费用。
