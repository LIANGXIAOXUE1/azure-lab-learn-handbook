
# Azure OpenAI 模型部署基础

## 1. 是什么

Azure OpenAI 是 Azure 上提供的大模型能力服务，可以让你在 Azure 平台中使用类似聊天、文本生成、总结、问答、代码辅助等 AI 能力。

你可以先把它理解成：

> Azure 提供的大模型调用平台，你可以在自己的 Azure 环境里部署并使用模型能力。

这里要特别注意一个词：

- **部署（Deploy）**

在 Azure OpenAI 里，很多时候不是“直接选一个模型就开始用”，而是要先在 Azure 资源里创建模型部署（Deployment），然后应用程序再调用这个部署。

所以学习 Azure OpenAI 的第一步，不只是知道“有模型”，而是理解：

> 你需要先创建 Azure OpenAI 资源，再在资源里部署模型，最后才能测试和调用。

---

## 2. 解决什么问题

Azure OpenAI 主要解决下面这些问题：

### 2.1 让企业在 Azure 上使用大模型能力
例如：

- 聊天问答
- 文本生成
- 总结摘要
- 分类提取
- 内容改写
- 代码辅助
- 企业知识库问答

### 2.2 方便与 Azure 其他服务组合
例如和这些服务配合：

- Blob Storage
- Azure AI Search
- Speech
- Key Vault
- App Service
- Functions

所以 Azure OpenAI 的价值通常不只是“单独聊天”，而是更容易进入企业应用场景。

### 2.3 满足企业对平台、权限、网络和治理的要求
很多企业关心的不只是“模型能不能用”，还包括：

- 谁可以访问
- 怎么管理密钥
- 怎么接入现有系统
- 怎么做网络和权限控制
- 怎么纳入 Azure 资源管理体系

## 3. 学习阶段先理解什么

学习 Azure OpenAI 时，建议先把下面这几件事分清楚。

### 3.1 Azure OpenAI 不是“ChatGPT 官网”
虽然它们底层能力方向相关，但使用方式不同。

在 Azure OpenAI 中，你通常会遇到这些 Azure 侧概念：

- Resource
- Region
- Model
- Deployment
- Endpoint
- Key

所以它更像是：

> 企业在 Azure 中使用大模型能力的一种平台化方式。

---

### 3.2 模型不等于部署
这是最重要的入门认知之一。

例如你可能看到一个模型名称，但这不代表你已经可以直接调用。  
通常还要先做一步：

- 创建 deployment（部署）

也就是说：

- **Model**：你要使用的模型类型
- **Deployment**：你在 Azure OpenAI 资源里实际创建的可调用实例名

很多初学者第一次调用失败，就是因为把模型名和部署名混了。

---

### 3.3 调用时常常需要这几个信息
后面无论你是用 Playground、SDK 还是代码调用，通常都会涉及：

- Endpoint
- API Key
- Deployment Name
- API Version

学习阶段先记住：

> 不是只知道“我用了哪个模型”就够了，还要知道“我部署成了什么名字”。

---

### 3.4 Azure OpenAI 有区域和配额限制
并不是所有区域都支持所有模型，也不是所有订阅默认都有完整权限。

学习阶段常见现象包括：

- 某个区域不能创建资源
- 某个模型不可用
- 某个模型无法部署
- 权限未开通
- 配额不足

所以 Azure OpenAI 的学习，往往比普通 VM 或 Storage 更容易遇到“资源存在但模型不可用”的情况。

---

## 4. Azure OpenAI 最基础的使用流程

学习阶段可以先把整体流程理解成下面这样：

1. 创建 Azure OpenAI 资源
2. 进入 Azure AI Foundry / Azure OpenAI Studio（具体入口可能会变化）
3. 选择一个可用模型
4. 创建 Deployment
5. 在 Playground 中测试
6. 获取 Endpoint / Key / Deployment Name
7. 用代码或应用接入

这个顺序很重要，因为很多初学者会跳过“部署”这一步，导致后面调用时报错。

---

## 5. 怎么创建和测试

下面以学习实验为目标，讲最基础的 Azure OpenAI 模型部署流程。

---

### 5.1 前置准备

建议先确认下面这些条件：

- 你有可用的 Azure Subscription
- 当前订阅已具备 Azure OpenAI 相关权限
- 所选 Region 支持 Azure OpenAI
- 资源组已创建好，例如：

```text
rg-azure-learning-openai
```

---

### 5.2 创建 Azure OpenAI 资源

1. 登录 Azure Portal
2. 搜索 Azure OpenAI 相关资源入口
3. 点击 `Create`
4. 选择：
   - Subscription
   - Resource Group
   - Region
5. 输入资源名称，例如：

```text
aoai-learn-01
```

6. 完成创建

> 注意：不同时间点 Azure Portal 的入口名称和页面可能略有变化，但核心思路基本一致。

---

### 5.3 进入资源并打开模型管理 / Studio 入口

创建完成后，通常会进入 Azure OpenAI 资源详情页。  
然后你会看到类似入口：

- Explore Azure AI Foundry
- Open in Studio
- Deployments
- Models

不同界面版本名称可能不同，但你要找的核心内容是：

- 模型可用列表
- 部署入口
- Playground 测试入口

---

### 5.4 创建模型 Deployment

这一步非常关键。

通常你需要：

1. 选择一个可用模型
2. 输入 Deployment Name，例如：

```text
gpt-4o-mini-test
```

3. 确认部署

> 这里的 Deployment Name 是你后续实际调用时经常要用到的名字，不一定等于模型原始名称。

---

### 5.5 使用 Playground 测试

部署完成后，进入 Playground，做一个最基础测试。

例如输入：

```text
请用一句话介绍 Azure OpenAI 是什么。
```

如果模型成功返回内容，说明最基础的资源、部署和调用链路已经通了。

---

### 5.6 查看调用所需信息

学习阶段建议记录下面这些关键内容：

- Resource Name
- Region
- Endpoint
- Deployment Name
- API Key
- 使用的模型类型

后面无论是写代码还是做客户演示，这些信息都非常重要。

---

## 6. 怎么测试

学习 Azure OpenAI，建议至少做下面几组基础测试。

---

### 6.1 测试 1：确认资源创建成功

检查：

- Azure OpenAI 资源是否部署成功
- Region 是否正确
- 是否能进入资源详情页

---

### 6.2 测试 2：确认模型可部署

检查：

- 目标模型是否在当前区域可见
- 是否允许创建 deployment
- deployment 是否创建成功

---

### 6.3 测试 3：Playground 基础问答

例如测试：

- 普通问答
- 一句话总结
- 简单改写
- 中英文转换

如果 Playground 能正常响应，说明基础能力已打通。

---

### 6.4 测试 4：记录部署名并确认自己没有混淆

建议你明确记录：

- 模型名是什么
- 部署名是什么

例如：

- 模型：`gpt-4o-mini`
- 部署名：`gpt-4o-mini-test`

这是后续非常容易出错的地方。

---

## 7. 这个实验想让你理解什么

Azure OpenAI 模型部署基础，不只是“把一个模型点出来”，更重要的是理解下面这些事。

### 7.1 Azure OpenAI 的使用是“资源 + 模型 + 部署”的组合
不是单独知道模型名就可以直接调用。

### 7.2 部署名是调用链路中的关键对象
后面很多报错，不是模型不行，而是部署名写错了。

### 7.3 Playground 是最适合入门验证的地方
先在 Playground 验证成功，再去写代码，会比一上来就写 API 更稳定。

### 7.4 区域、权限、配额是 Azure OpenAI 学习中的高频限制点
如果资源和模型有问题，不要先怀疑代码，先看区域和权限。

## 8. 常见问题

### 问题 1：为什么我创建了 Azure OpenAI 资源，但看不到想要的模型？

常见原因包括：

- 当前 Region 不支持该模型
- 当前订阅权限不足
- 该模型当前未开放
- 配额或访问资格有限制

#### 排查建议
先检查：

- 当前资源所在 Region
- 目标模型在该区域是否支持
- 当前订阅是否具备访问资格

---

### 问题 2：为什么 Playground 可以用，但代码调用失败？

常见原因包括：

- Endpoint 写错
- API Key 错误
- Deployment Name 写错
- API Version 不匹配
- 调用格式不正确

#### 排查建议
先核对：

- Endpoint
- Key
- Deployment Name
- API Version

尤其是 Deployment Name，不要只写模型名。

---

### 问题 3：模型名和部署名有什么区别？

可以简单理解为：

- **模型名**：平台提供的模型类型
- **部署名**：你在自己资源里创建的调用入口名字

很多时候代码调用用的是部署名，而不是你看到的原始模型名称。

---

### 问题 4：为什么部署创建失败？

常见原因包括：

- 当前区域不支持该模型
- 资源权限不足
- 配额不足
- 模型当前不可部署

---

### 问题 5：Azure OpenAI 和 Azure AI Services 是一回事吗？

不是完全一回事。

学习阶段可以先简单理解为：

- **Azure OpenAI**：更偏大模型、聊天、生成式 AI 能力
- **Azure AI Services**：更偏语音、翻译、视觉、文档识别等预构建 AI 能力集合

两者可以组合使用，但定位不完全相同。

## 9. 客户常问的问题

这一部分重点不是只记录客户怎么问，而是练习你该怎么回答。

---

### 问题 1：Azure OpenAI 是什么？

**简短回答：**  
Azure OpenAI 是 Azure 上提供的大模型能力服务，支持聊天、文本生成、总结、问答等生成式 AI 场景。

**展开回答：**  
如果客户关心的是企业怎么在 Azure 上使用大模型能力，例如智能问答、内容生成、知识库问答、文本处理等，Azure OpenAI 就是非常核心的服务之一。  
它的价值不只是“能聊天”，而是更容易接入企业现有系统和 Azure 服务体系。

---

### 问题 2：是不是创建资源后就能直接调用？

**简短回答：**  
不一定，通常还需要先做模型部署（Deployment）。

**展开回答：**  
很多初学者会以为资源创建好就能直接调模型。  
但在 Azure OpenAI 中，通常还要先把可用模型部署成一个 deployment，后面调用时再使用这个 deployment name。

---

### 问题 3：是不是一定要自己训练模型？

**简短回答：**  
不一定，很多场景下并不需要自己训练模型。

**展开回答：**  
企业常见需求，比如问答、总结、改写、知识库增强，很多时候更多是：

- 选模型
- 做提示词设计
- 接企业数据
- 做权限和应用集成

而不是从零训练一个模型。

---

### 问题 4：Azure OpenAI 能不能做企业知识库？

**简短回答：**  
可以，但通常不是只靠 Azure OpenAI 一个服务，而是要和检索、存储、应用层一起组合。

**展开回答：**  
最常见的组合方向包括：

- Azure OpenAI
- Azure AI Search
- Blob Storage
- App Service / Functions
- 身份权限控制

所以客户问“能不能做知识库”，答案通常是能，但这是一个组合方案，不只是“开一个模型”就结束。

---

### 问题 5：Azure OpenAI 安全吗？

**简短回答：**  
Azure 提供了企业级的平台能力，但是否安全还要看访问控制、网络、密钥管理和应用架构设计。

**展开回答：**  
客户真正关心的通常包括：

- 谁能调用
- Key 怎么管理
- 数据怎么接入
- 是否有权限边界
- 是否能做私有网络
- 是否有审计和治理能力

所以“平台有能力”不等于“默认就天然安全”，还是要结合整体架构来看。

- Azure OpenAI 的使用通常是资源、模型、部署三者结合
