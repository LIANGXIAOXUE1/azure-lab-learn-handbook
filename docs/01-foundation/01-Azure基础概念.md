# Azure 基础概念

## 1. 是什么

Azure 是微软提供的云计算平台（Cloud Platform），提供计算、存储、网络、数据库、安全、监控、AI、数据分析、物联网等多种云服务。

简单理解：

Azure 就是把原来需要企业自己购买、部署、维护的服务器、网络、存储、数据库等基础能力，放到云上，以服务的形式按需使用。

你不再需要先准备机房、采购硬件、安装环境，很多能力都可以直接在 Azure 上快速开通和使用。

---

## 2. 解决什么问题

Azure 主要解决的是企业 IT 和业务系统上线过程中常见的几个问题：

### 2.1 不想自己买服务器和机房
传统方式需要自己采购服务器、搭建机房、处理电力、网络、备份、扩容和维护。  
Azure 把这些底层基础设施托管起来，用户可以直接使用。

### 2.2 希望更快上线业务
如果要部署网站、API、数据库、内部系统，使用 Azure 可以比本地采购和部署更快。

### 2.3 希望按需扩容
业务流量小的时候少用资源，业务流量大时再扩容，不需要一开始就买很多硬件。

### 2.4 希望支持全球部署
Azure 在多个区域（Region）提供服务，可以把业务部署到更接近客户的位置。

### 2.5 希望提升运维、安全、监控能力
Azure 提供监控、日志、告警、身份认证、权限控制、备份、灾备等能力，方便统一管理。

### 2.6 希望直接使用 AI 和高级云服务
例如：

- Azure OpenAI
- Speech
- AI Search
- Document Intelligence
- Translation

这些都可以直接调用，而不需要从零开始搭建整套能力。

## 3. 核心理解：Azure 不只是“云服务器”

初学 Azure 时，最容易产生一个误解：

> 以为 Azure 只是“把服务器搬到云上”。

其实不是。  
虚拟机（Virtual Machine, VM）只是 Azure 中最基础的一类服务。Azure 更重要的价值在于，它提供的是一整套云平台能力。

常见能力包括：

- **计算**：Virtual Machine、App Service、AKS、Functions
- **存储**：Blob Storage、Disk、Files
- **网络**：VNet、Subnet、Load Balancer、Application Gateway、DNS
- **数据库**：Azure SQL、MySQL、PostgreSQL、Cosmos DB
- **身份与安全**：Microsoft Entra ID、RBAC、Key Vault
- **监控运维**：Azure Monitor、Log Analytics、Alerts
- **AI 服务**：Azure OpenAI、AI Services、Speech、AI Search
- **数据与集成**：Data Factory、Event Hub、Service Bus

所以更准确地说：

> Azure 是一个云平台，而不是单一的云服务器产品。

## 4. Azure 的基础资源模型

在真正开始创建资源之前，需要先理解几个最基础的概念。

### 4.1 Subscription（订阅）
Subscription 是 Azure 中计费和资源管理的基本边界。  
你创建的大部分资源，都属于某个订阅。

可以简单理解为：

- 它决定资源记在哪个账单下面
- 它也是权限和配额的重要边界之一

### 4.2 Resource Group（资源组）
Resource Group 是资源的逻辑分组容器。  
通常会把同一个项目、同一套实验、同一套应用相关的资源放在同一个资源组里。

例如：

- 一个测试环境一个资源组
- 一个项目一个资源组
- 一个练习实验一个资源组

### 4.3 Region（区域）
Region 是 Azure 数据中心所在的地理区域，例如：

- East US
- West Europe
- Southeast Asia

创建资源时通常要选择区域。  
区域会影响：

- 延迟
- 服务可用性
- 合规要求
- 成本
- 某些服务是否支持

### 4.4 Resource（资源）
在 Azure 中，虚拟机、存储账号、数据库、虚拟网络、公网 IP 等都属于资源。

也就是说，你在 Azure Portal 里创建的每一样东西，本质上都是一个 Azure Resource。

## 5. 怎么开始使用

最基本的使用步骤如下：

1. 注册 Azure 账号
2. 拥有一个可用的 Subscription（订阅）
3. 登录 Azure Portal
4. 创建一个 Resource Group（资源组）
5. 在资源组中创建所需资源，例如：
   - Virtual Machine
   - Storage Account
   - Virtual Network
   - SQL Database

常见操作入口包括：

### 5.1 Azure Portal
图形化界面操作，适合初学者快速理解资源关系和配置过程。

### 5.2 Azure CLI
命令行操作，适合批量管理、自动化脚本和日常运维。

### 5.3 PowerShell
适合习惯微软生态的管理员进行自动化操作。

### 5.4 ARM / Bicep / Terraform
用于基础设施即代码（IaC），适合标准化部署和团队协作。

## 6. 怎么测试

学习 Azure 基础概念时，不需要一开始就创建很多复杂资源，先完成最简单的基础测试即可。

建议测试步骤如下：

### 6.1 登录 Azure Portal
确认可以正常访问 Azure Portal，并看到自己的订阅信息。

### 6.2 查看当前 Subscription
确认当前账号下是否有可用订阅。

### 6.3 创建一个 Resource Group
例如创建：

- 名称：`rg-azure-learning-test`
- 区域：`East Asia` 或你当前可用区域

### 6.4 查看资源组概览页面
确认能够进入资源组详情页面，查看位置、标签、资源数量等信息。

### 6.5 删除测试资源组
删除这个测试资源组，理解 Azure 资源是可以创建、查看、修改、删除的。

如果你能完成以上动作，说明你已经完成了 Azure 最基础的入门。

## 7. 学习 Azure 时最先要建立的认知

初学 Azure，不建议一开始就试图把所有服务都学完。  
更重要的是先建立下面这些基础认知：

### 7.1 Azure 是资源化管理的
你做的大部分事情，本质上都是在“创建、配置、连接、删除资源”。

### 7.2 Azure 是按服务组合来解决问题的
一个完整方案通常不是只靠一个服务，而是多个服务组合：

- App Service + SQL Database + Key Vault
- VM + VNet + NSG + Public IP
- Blob Storage + SAS + CDN
- Azure OpenAI + AI Search + Speech

### 7.3 Azure 学习不只是“会点 Portal”
真正有价值的是你能回答这些问题：

- 这个服务是什么？
- 它解决什么问题？
- 什么时候该用它？
- 跟别的服务有什么区别？
- 出问题时先查哪里？

## 8. 常见问题

### 问题 1：Azure 是不是只有虚拟机？
不是。  
虚拟机只是 Azure 最基础的一类计算服务。Azure 还提供应用托管、容器、数据库、存储、网络、安全、监控、AI 等大量服务。

例如：

- 网站托管：App Service
- 数据库：Azure SQL Database
- 对象存储：Blob Storage
- AI：Azure OpenAI、Speech、AI Search

---

### 问题 2：Azure 是不是必须会命令行？
不是。  
初学阶段完全可以先通过 Azure Portal 学习核心概念，例如：

- 订阅
- 资源组
- 区域
- 虚拟机
- 存储
- 网络

等理解了资源关系后，再学习 Azure CLI、PowerShell、Bicep 或 Terraform 会更顺。

---

### 问题 3：Azure 和本地服务器的区别是什么？
本地服务器通常需要自己采购、部署、维护硬件和环境；Azure 则是云上按需使用。

可以简单对比为：

- **本地服务器**：自己买、自己装、自己维护
- **Azure**：按需开通、快速部署、平台托管更多底层能力

Azure 的优势通常体现在：

- 上线快
- 扩容快
- 全球部署方便
- 监控和安全能力更完整
- 更适合与云服务组合

---

### 问题 4：用了 Azure 以后，是不是就完全不用管运维了？
不是。  
Azure 能帮你托管很多底层基础设施，但并不代表完全没有运维工作。

比如你使用 VM 时，仍然需要关注：

- 操作系统
- 端口开放
- 用户账号
- 补丁更新
- 应用部署
- 安全配置

只是相比传统本地环境，Azure 可以让很多基础工作更标准化、更自动化。

---

### 问题 5：Azure 是不是很贵？
不一定。  
Azure 是按使用量和服务规格计费，不同架构成本差异很大。

影响成本的因素包括：

- 资源规格大小
- 使用时长
- 存储容量
- 网络流量
- 区域
- 是否有备份、监控、高可用等配置

所以不能简单说“Azure 一定贵”或者“一定便宜”，要看具体场景和方案设计。

## 9. 客户常问的问题

这一部分不是只记录“客户会怎么问”，还要练习“你应该怎么答”。

---

### 问题 1：Azure 能做什么？

**简短回答：**  
Azure 可以提供企业上云所需的大部分基础能力，包括服务器、存储、网络、数据库、安全、监控，以及 AI、数据分析、物联网等服务。

**展开回答：**  
如果客户想部署网站、API、数据库、内部系统、文件存储、AI 能力，Azure 都有对应服务。  
它不是一个单一产品，而是一整套云平台。

---

### 问题 2：我们把系统放到 Azure 上有什么好处？

**简短回答：**  
主要好处是部署更快、扩展更灵活、全球覆盖更方便，同时更容易获得监控、安全、备份和身份管理能力。

**展开回答：**  
相比自己买服务器，Azure 更适合业务变化快、希望快速上线、希望提升稳定性和管理效率的场景。  
如果后续还要接 AI、日志、监控、身份管理，也更容易做集成。

---

### 问题 3：一定要买服务器吗？

**简短回答：**  
不一定。  
如果只是部署网站、API 或常规业务系统，很多时候不需要自己管理服务器，可以直接用平台服务。

**展开回答：**  
比如：

- 想托管网站或 API，可以考虑 **App Service**
- 想运行容器，可以考虑 **AKS** 或其他容器服务
- 想执行事件驱动代码，可以考虑 **Functions**
- 只有在确实需要操作系统级控制时，才更适合使用 **VM**

所以“上云”不等于“一定买云服务器”。

---

### 问题 4：Azure 能不能部署网站、接口、数据库？

**简短回答：**  
可以，而且这是 Azure 最常见的使用方式之一。

**展开回答：**  
常见组合方式：

- 网站 / API：App Service 或 VM
- 数据库：Azure SQL、MySQL、PostgreSQL
- 文件或对象存储：Blob Storage
- 安全密钥管理：Key Vault

所以无论是传统 Web 系统还是现代 API 架构，Azure 都能支持。

---

### 问题 5：Azure 上能不能做 AI？

**简短回答：**  
可以，而且 Azure 提供的是比较完整的企业级 AI 能力。

**展开回答：**  
例如：

- 大模型能力：Azure OpenAI
- 语音识别与语音合成：Speech
- 文档识别：Document Intelligence
- 企业搜索：AI Search
- 翻译：Translation

如果客户关心的是“企业知识库”“智能问答”“语音”“OCR”“文档处理”等，Azure 都有对应方案方向。

---

### 问题 6：数据放 Azure 安全吗？

**简短回答：**  
Azure 提供了比较完整的安全能力，但是否安全还取决于实际架构设计、权限控制和配置方式。

**展开回答：**  
Azure 提供很多安全相关能力，例如：

- 身份认证
- 权限控制
- 私有网络访问
- 密钥管理
- 日志审计
- 数据加密

但“云平台本身有安全能力”不等于“你的系统天然就是安全的”。  
安全还需要结合：

- 账号权限设计
- 网络访问控制
- 密钥管理方式
- 资源配置规范
- 日志与审计机制

来综合看。

## 10. 你可以怎么回答客户

如果客户只是第一次了解 Azure，建议回答时不要一开始就堆很多产品名，而是先讲成客户能听懂的话。

### 一种适合初次沟通的回答方式

> Azure 是微软的云平台，可以理解成把原来企业自己维护的服务器、存储、网络、数据库这些能力搬到云上，并且还能直接使用监控、安全、备份、AI 等服务。  
> 对客户来说，价值通常不是“多了一个云服务器”，而是系统上线更快、扩展更灵活、管理更方便，也更容易和数据库、安全、AI 服务组合起来。

这类回答的重点是：

- 先讲价值
- 再讲能力
- 最后再根据客户场景展开到具体服务
