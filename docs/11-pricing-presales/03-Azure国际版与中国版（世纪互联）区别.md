# Azure 国际版与中国版（世纪互联）区别

这篇文档主要解决 Azure 报价里一个非常关键、但新手特别容易混淆的问题：

> **客户到底用的是 Azure Global（国际版），还是 Azure 中国版（由世纪互联运营的 Azure）？**

如果这一步判断错了，后面的报价通常会一路错下去，比如：

- 门户地址看错
- 区域看错
- 单价查错
- 优惠方式搞混
- 美元 / 人民币搞混
- 可用服务判断错误

所以在正式做任何报价前，必须先分清：

> **这次项目是国际版 Azure，还是中国版 Azure。**

---

# 一、先纠正一个常见叫法：是“世纪互联”，不是“世纪莲”

中国大陆境内合规运营的 Azure，正确名称是：

> **由世纪互联运营的 Azure**

常见简称有：

- Azure 中国
- Azure 中国版
- 21V Azure
- 世纪互联 Azure

而境外的 Azure，一般称为：

- Azure Global
- Azure 国际版
- Azure 全球版

这两套不是“同一个后台换个区域”这么简单，而是：

> **两套相互独立的云平台。**

---

# 二、一句话讲清楚两者区别

## Azure Global（国际版）
微软全球直营，面向海外区域，使用全球 Azure 门户与全球计费体系。

## Azure 中国版（世纪互联运营）
微软授权世纪互联在中国大陆独立运营，面向中国大陆本地合规需求，使用独立门户与独立计费体系。

---

# 三、最核心结论：两者完全隔离

这是你做报价时必须死记住的一条：

> **Azure Global 和 Azure 中国版，在账号、订阅、计费、网络、区域、价格、优惠方式上都是分开的。**

你可以直接理解为：

- 不是一个账号体系
- 不是一个价格体系
- 不是一个门户
- 不是一个资源池
- 不是一个可直接迁移的环境

也就是说：

## 不能互通登录
- `portal.azure.com` 的账号，不能直接登录 `portal.azure.cn`
- `portal.azure.cn` 的账号，也不能直接当成国际版账号用

## 不能直接混用报价
- 国际版美东价格，不能拿去给中国北部 3 报价
- 中国版人民币价格，也不能拿去给美国东部报价

## 不能默认服务都一样
- 有些国际版服务，中国版未必有
- 有些国际版区域支持的产品，中国版可能不支持

---

# 四、报价最实用的理解方式：先把两者当成“两套云”

对于新手来说，最省事的理解方式就是：

> **把 Azure Global 和 Azure 中国版，先当成两套独立云来处理。**

这样你就不容易犯错。

---

# 五、一张表看懂两者核心差异

| 对比项 | Azure Global 国际版 | Azure 中国版（世纪互联运营） |
| ---- | ---- | ---- |
| 运营主体 | 微软直接运营 | 世纪互联独立运营 |
| 常见叫法 | 国际版 / 全球版 Azure | Azure 中国 / 世纪互联 Azure |
| 门户地址 | `portal.azure.com` | `portal.azure.cn` |
| 计价货币 | 美元 USD | 人民币 CNY |
| 常见优惠方式 | RI（预留实例，1年/3年） | CPP（承诺套餐） |
| 区域范围 | 美国、欧洲、东南亚、日本等全球区域 | 中国大陆本地区域 |
| 数据驻留 | 海外数据中心 | 中国大陆境内数据中心 |
| 账号体系 | 全球 Azure 账号体系 | 中国独立账号体系 |
| 适合客户 | 跨境业务、海外官网、海外 APP、国际化系统 | 国内政企、纯国内业务、数据不出境项目 |
| 报价习惯 | 美元报价，必要时换算人民币 | 直接人民币报价 |
| 你要查的价格页 | Azure Global 定价页 / 计算器 | Azure 中国官网价格页 |
| 服务可用性 | 更完整 | 部分服务可能不同或受限 |

# 六、你做报价时，为什么一定要先分清这两套

因为这不是“名称差异”，而是会直接影响你整个报价逻辑。

---

## 1）价格体系不同

同样一个机型，比如 D8 系列：

- 国际版是国际版价格
- 中国版是中国版价格

两边不共用同一套单价。

所以：

> **不能拿国际版价格去报中国版，也不能拿中国版价格去报国际版。**

---

## 2）货币不同

### Azure Global
通常是：
- 美元计价
- 最终可能需要按汇率换算成人民币展示

### Azure 中国版
通常是：
- 人民币计价
- 直接本币报价

所以你做报价表时，列名和汇率处理都不一样。

---

## 3）优惠方式不同

### Azure Global 常见优惠
- Pay as you go（按量）
- RI（Reserved Instance，预留实例）
- 可能还有其他全球优惠政策

### Azure 中国版常见优惠
- 按量
- CPP（承诺套餐）

所以你不能把两边的优惠列头写混。

例如：

### 国际版报价常见列
- 按量月付
- RI 1 年价格
- RI 3 年价格（如适用）

### 中国版报价常见列
- 按量
- CPP 优惠价

---

## 4）区域不同

### 国际版常见区域
- East US
- Central US
- West Europe
- Southeast Asia
- Japan East
- France Central

### 中国版常见区域
- 中国北部 3
- 中国东部 2
等中国大陆区域

如果客户明确说：
- 美国东部
- 法国
- 新加坡
- 日本

那基本就是国际版思路。

如果客户明确说：
- 中国北部 3
- 中国东部 2
- 数据必须落国内

那基本就是中国版思路。

---

## 5）服务可用性不同

并不是国际版有的所有服务，中国版都有。

你做报价时要特别注意：

- 某些 AI 服务可能不同
- 某些安全服务可能不同
- 某些加密、密钥、HSM、分析类产品可能有差异
- 某些区域不支持某些 SKU

所以不能机械地把国际版架构照搬到中国版。

---

# 七、怎么最快判断客户到底是哪一套

这是最实用的部分。

你可以在报价前，直接问销售或客户下面 3 个问题。

---

# 八、交接必问的 3 个判断问题

---

## 问题 1：业务部署在哪个地区

你可以这样问：

> 这套系统是部署在海外区域，还是部署在中国大陆境内？

### 如果答案是这些
- 美国
- 法国
- 欧洲
- 新加坡
- 日本
- 香港
- 海外用户访问为主

通常按：

> **Azure Global（国际版）**

### 如果答案是这些
- 中国大陆机房
- 数据不出境
- 要满足国内合规
- 部署在中国北部 3 / 中国东部 2

通常按：

> **Azure 中国版（世纪互联运营）**

---

## 问题 2：客户登录的是哪个门户

这是一个非常直接的判断方法。

你可以问：

> 客户平时登录 Azure 的后台网址是什么？

### 如果是
- `portal.azure.com`

那通常是：

> **Azure Global**

### 如果是
- `portal.azure.cn`

那通常是：

> **Azure 中国版（世纪互联运营）**

---

## 问题 3：客户结算是美元还是人民币

你可以问：

> 客户这次项目是按美元结算，还是按人民币对公结算？

### 如果是
- 美元
- 海外主体
- 走 RI

大概率是：

> **Azure Global**

### 如果是
- 人民币
- 国内主体
- 走 CPP / 国内套餐

大概率是：

> **Azure 中国版**

---

# 九、再给你一个最简单判断法：看“业务在哪里”和“数据放哪里”

如果你还不熟，先记这个最简单的办法。

## 海外业务 / 海外部署 / 全球访问
优先想到：

> **Azure Global**

## 纯国内业务 / 数据不能出境 / 国内合规
优先想到：

> **Azure 中国版（世纪互联运营）**

---

# 十、两种典型客户场景，直接判断给你看

---

## 场景 1：跨境穿戴设备 APP，服务器部署北美

### 信息
- 面向海外用户
- 部署在美国
- 海外 APP 后台
- 通常美元结算

### 判断
这类场景通常用：

> **Azure Global 国际版**

### 报价特点
- 查国际版价格
- 关注美东 / 美中等海外区域
- 用美元计价
- 可做 RI 方案

---

## 场景 2：国内制造企业 OA 系统，要求数据不出境

### 信息
- 只给国内员工使用
- 数据必须落在国内
- 可能涉及等保 / 审计 / 合规
- 通常人民币结算

### 判断
这类场景通常用：

> **Azure 中国版（世纪互联运营）**

### 报价特点
- 查中国版价格
- 区域看中国北部 / 中国东部
- 用人民币计价
- 关注 CPP 等国内优惠方式

---

# 十一、对你做报价的直接影响

下面这些是你在实际工作里最容易用到的。

---

## 1）报价前第一步就要先判断版本

建议你以后拿到任何需求，先在最上面写一行：

- Azure Global（国际版）
或
- Azure 中国版（世纪互联运营）

这样后面不容易全程搞混。

---

## 2）报价表列名要跟版本一致

### 国际版常见写法
- Pay as you go
- RI 1 Year
- RI 3 Year
- USD / RMB（如做汇率换算）

### 中国版常见写法
- 按量
- CPP
- 人民币总价

不要把：
- RI
- CPP
- USD
- CNY

这些混着写。

---

## 3）国际版才需要重点考虑汇率换算

如果你内部模板是：

- 先按美元出
- 再按固定汇率转人民币展示

那这是典型国际版思路。

而中国版本身就是人民币价，一般不需要做美元换算。

---

## 4）同一机型不能默认价格接近

很多新手容易误以为：

> 同样都是 D8，价格应该差不多吧？

不能这样想。

因为两边：
- 运营体系不同
- 区域不同
- 价格体系不同
- 优惠方式不同

所以报价时要完全按对应版本的价格源来查。

---

## 5）要额外关注中国版是否支持对应服务

如果客户需求里有这些内容，更要多留意：
- AI 服务
- 安全高级功能
- 加密 / HSM
- 某些分析服务
- 某些区域专属能力

国际版可用，不代表中国版一定可用。

---

# 十二、报价时最容易犯的 6 个错误

---

## 错误 1：把 Azure 中国版和国际版当成同一个平台

这是最根本的错误。

它们不是同一个租户下的不同区域，而是两套独立体系。

---

## 错误 2：用国际版价格去报中国版

这是报价里最致命的问题之一。

一旦价格源错了，整张报价单基本就失效了。

---

## 错误 3：客户明明是 portal.azure.cn，你还去查 global 价格

所以门户地址是一个非常好用的判断依据。

---

## 错误 4：把 RI 和 CPP 混着写

这会让报价显得非常不专业，也容易让销售和客户混淆。

---

## 错误 5：默认服务完全一致

中国版不是国际版的“镜像复制版”，不能默认每项服务都完全一样。

---

## 错误 6：明明是国内合规项目，却推荐海外区域

如果客户有明确的数据驻留和合规要求，部署区域不是成本问题，而是前提条件问题。

---

# 十三、建议你以后在需求收集表里新增这 1 项

为了避免后面反复确认，建议你在需求收集模板里加一个字段：

## 云环境类型
- Azure Global（国际版）
- Azure 中国版（世纪互联运营）
- 待确认

这样后面你拿到需求时，会更清晰。

# 十四、给销售可直接复制的话术

下面这段你可以直接发给销售确认：

---

## Azure 环境确认信息

请先协助确认本次项目使用的是哪一套 Azure 环境：

1. 客户当前登录 Azure 后台的网址是：
   - `portal.azure.com`
   - `portal.azure.cn`

2. 本次业务部署区域是：
   - 海外区域（如美国、欧洲、新加坡、日本等）
   - 中国大陆区域（如中国北部 3、中国东部 2 等）

3. 本次结算方式是：
   - 美元 / 国际版
   - 人民币 / 中国版

4. 是否有数据不出境、国内合规、等保等要求：
   - 是
   - 否

如以上任一项涉及中国大陆本地部署或 portal.azure.cn，请按 Azure 中国版处理；  
如为海外区域部署或 portal.azure.com，请按 Azure Global 处理。

# 十五、这篇你要记住的核心结论

## 1. Azure Global 和 Azure 中国版是两套独立体系
不是一个平台里的不同区域。

## 2. 报价前必须先判断客户属于哪一套
否则后面价格、区域、服务和优惠方式都会错。

## 3. 最好用 3 个问题快速判断
- 部署区域
- 门户地址
- 结算货币

## 4. 国际版常见是美元 + RI
中国版常见是人民币 + CPP

## 5. 两边服务和价格都不能直接混用
一定要按对应版本单独查价和出方案。
